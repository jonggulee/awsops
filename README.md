# awsops

k9s처럼 터미널에서 동작하는 AWS 멀티 어카운트 리소스 뷰어.

## 배경

회사에서 여러 AWS 어카운트에 걸쳐 리소스를 관리하다 보면, 어카운트마다 콘솔을 열거나 CLI를 따로 실행해야 하는 불편함이 있다. [cloudlens](https://github.com/one2nc/cloudlens) 같은 툴이 있지만 단일 어카운트만 지원한다. awsops는 여러 어카운트의 리소스를 하나의 터미널 화면에서 한눈에 볼 수 있게 만드는 것이 목표다.

## 목표

- **Read-only**: 조회만 한다. 생성/삭제/수정 없음
- **멀티 어카운트**: `~/.aws/config`에 설정된 여러 프로필을 동시에 조회
- **TUI**: 터미널 UI로 동작 (k9s 스타일)

## 기술 스택

- **언어**: Go
- **TUI**: [bubbletea](https://github.com/charmbracelet/bubbletea) (Elm 아키텍처)
- **AWS SDK**: aws-sdk-go-v2

## 기능 로드맵

### v1 - 기본 뷰어
- [ ] 여러 AWS 프로필(어카운트)의 EC2 인스턴스를 한 화면에 조회
- [ ] 리소스 타입 검색 (ec2, s3, lambda 등 입력하면 해당 리소스 목록 표시)
- [ ] 어카운트 필터링 (특정 프로필만 보기)
- [ ] 리전 필터링 (특정 리전만 보기)

### v2 - 고도화
- [ ] 리소스간 관계 시각화 (EC2 ↔ Security Group ↔ VPC)
- [ ] 태그 기반 검색 및 필터링
- [ ] 리소스 상태 실시간 갱신 (polling)
- [ ] 어카운트간 리소스 diff 비교

### v3 - 심화
- [ ] 비용 정보 오버레이 (Cost Explorer 연동)
- [ ] 즐겨찾기/북마크 리소스
- [ ] 설정 파일로 어카운트/리전 프리셋 관리

## 사용법

```bash
# 빌드
go build -o awsops .

# 실행
./awsops
```

## 키 바인딩 (예정)

| 키 | 동작 |
|---|---|
| `/` | 리소스 검색 |
| `f` | 필터 (어카운트/리전) |
| `r` | 새로고침 |
| `q` | 종료 |
| `↑↓` | 목록 이동 |
| `Enter` | 상세 보기 |
